## 2. 要件定義を最後にする理由

### 背景

- AI がバンバン開発できる時代になった
- 「まずは要件定義から」という従来のアプローチを最初に教えると、受講生がすぐに複雑な接続やシステム構築を始めてしまう

### この順序にする意図

1. **まず体験する** - チュートリアルで Claude Code を触って「AI で作れる」感覚を掴む
2. **環境を整えてもっと作る** - VS Code + スキルで効率アップ、Next.js で本格的なアプリ作成
3. **作ったものを管理する** - Git/GitHub でバージョン管理を学ぶ（作るものがあるから意味がわかる）
4. **大きなプロジェクト向けの手法として紹介** - 十分な基礎ができた段階で、要件定義ドリブンな開発手法を学ぶ

## 3. 各ガイドのポイント（講義時の補足）

### 1. チュートリアル

- 講義の最初にまず触らせる
- 「自然言語で指示するだけでコードが生成される」体験
- 細かい説明は後回し、まず「動いた！」を優先

### 2. VS Code の使い方

- 全員のエディタ環境を揃える
- ターミナル統合の便利さを強調
- 拡張機能は最小限に（後から追加していく方針）

### 3. Claude Code スキル

- CLAUDE.md の重要性を伝える
- プロジェクト固有のルールを AI に伝える方法
- スキル機能のカスタマイズは応用編として軽く触れる

### 4. Next.js アプリを作ろう

- Node.js のインストール確認から
- `npm run dev` で動くまでを確実に
- 「自分で作った！」の成功体験を大事に

### 5. Git の使い方

- 「セーブポイント」の比喩を使って説明
- 「さっき作ったアプリを保存しよう」という文脈で教える
- コミットは「こまめに」「意味のある単位で」

### 6. GitHub の使い方

- SSH 鍵の設定でつまずきやすいので丁寧に
- 「クラウドバックアップ」としての価値を伝える
- リポジトリは Public でもOK（練習用）

### 7. 環境変数を理解しよう

- API キーの扱い方は現場でも必須
- `.env` ファイルと `.gitignore` の関係
- 「秘密の情報は GitHub に上げない」を徹底

### 8. 外部サービスと連携しよう

- API の基本概念（リクエスト・レスポンス）
- 認証の種類（API キー、OAuth）
- 実際に動くものを見せながら説明

### 9. Neon DB + Prisma 接続ガイド

- データベースの概念が初めての人向けに補足
- Neon の無料枠で十分練習できる
- Prisma の型安全性のメリットを伝える

### 10. 要件定義書から始める開発

- 「なぜ要件定義が必要か」の背景説明
- 小さなプロジェクトではオーバーキル、大きなプロジェクトで威力を発揮
- Plan Mode の活用方法を実演


## 5. よくある質問・トラブル

### Q: Git の初期設定でエラーが出る
- `git config --global user.name` と `user.email` の設定を確認

### Q: GitHub に push できない
- SSH 鍵の設定を再確認
- リポジトリの URL が正しいか確認（HTTPS vs SSH）

### Q: npm install でエラーが出る
- Node.js のバージョンを確認（推奨: LTS版）
- `node_modules` を削除して再実行

### Q: 環境変数が読み込まれない
- `.env.local` のファイル名を確認
- `NEXT_PUBLIC_` プレフィックスの有無を確認
